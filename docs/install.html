New to LaTeX and VS Code? Follow the platform-specific steps below.

Download and Install

Requires Visual Studio Code. Download the latest VSIX and install.

Step 1 — Install LaTeX (no GUI)

We recommend the official TeX Live/MacTeX distributions without a GUI. Pick your platform below; you can switch anytime.

macOS (MacTeX/BasicTeX)

  1. Install BasicTeX (no GUI, smaller download) from CTAN (BasicTeX.pkg). Alternatively, full MacTeX from tug.org/mactex.
  2. Open Terminal and install required engines, tools, and packages with tlmgr: 
    sudo tlmgr update --self
        sudo tlmgr install latexmk xetex biber \
          datetime2 footmisc comment geometry fontspec \
          langsci-gb4e forest qtree tipa csquotes biblatex xparse \
          setspace enumitem ragged2e needspace placeins float longtable \
          tabularx array multirow makecell booktabs diagbox xcolor tcolorbox \
          caption glossaries-extra etoolbox ulem fancyhdr hyperref cleveref pdflscape
  3. Verify tools: 
    xelatex --version
        biber --version
        latexmk -v

Windows (TeX Live)

  1. Download and run the TeX Live installer from tug.org/texlive/windows.html (choose scheme without GUI editors).
  2. Open the “TeX Live Command Prompt” (from Start Menu) and install packages: 
    tlmgr update --self
        tlmgr install latexmk xetex biber \
          datetime2 footmisc comment geometry fontspec \
          langsci-gb4e forest qtree tipa csquotes biblatex xparse \
          setspace enumitem ragged2e needspace placeins float longtable \
          tabularx array multirow makecell booktabs diagbox xcolor tcolorbox \
          caption glossaries-extra etoolbox ulem fancyhdr hyperref cleveref pdflscape
  3. Verify tools: 
    xelatex --version
        biber --version
        latexmk -v

Linux

Recommended: Use the official TeX Live installer (current year) for the latest packages.

  1. Install TeX Live per Quick Install.
  2. Then install required packages with tlmgr: 
    tlmgr update --self
        tlmgr install latexmk xetex biber \
          datetime2 footmisc comment geometry fontspec \
          langsci-gb4e forest qtree tipa csquotes biblatex xparse \
          setspace enumitem ragged2e needspace placeins float longtable \
          tabularx array multirow makecell booktabs diagbox xcolor tcolorbox \
          caption glossaries-extra etoolbox ulem fancyhdr hyperref cleveref pdflscape
Alternatively, use your distro packages (quick, may be older)

Debian/Ubuntu

sudo apt update
        sudo apt install -y texlive texlive-latex-recommended texlive-latex-extra texlive-fonts-recommended texlive-xetex biber latexmk

Fedora

sudo dnf install -y texlive-scheme-medium texlive-biber latexmk

Arch

sudo pacman -S --needed texlive-most biber latexmk

If a package is missing at compile time, install it via your package manager or tlmgr where available.

Why these packages? Your LaTeX template uses XeLaTeX/LuaLaTeX and relies on: langsci-gb4e, graphicx, tabularx, biblatex (with biber), glossaries-extra, hyperref, cleveref, and many common formatting packages listed above.

Step 2 — Install Visual Studio Code and LaTeX extensions

Install VS Code, then add LaTeX helper extensions for the best experience:

  • VS Code: Download for Windows, macOS, Linux
  • LaTeX Workshop: Marketplace
  • LaTeX Utilities: Marketplace

    • Install the extensions from inside VS Code (Extensions view) or via CLI:

    code --install-extension James-Yu.latex-workshop
        code --install-extension tecosaur.latex-utilities

    Tip (macOS): enable the code shell command via Command Palette → “Shell Command: Install 'code' command in PATH”.

    Use XeLaTeX in LaTeX Workshop

    In VS Code: Command Palette → “LaTeX Workshop: Build with recipe” → choose “latexmk (xelatex)”. Ensure latexmk and xelatex are installed (Step 1).

    Prerequisite: Install Visual Studio Code

    LingTeX is a Visual Studio Code extension. Please install VS Code first:

    Step 3 — Install LingTeX from VSIX (GUI)

  • Windows, macOS, Linux: Get VS Code from code.visualstudio.com.
  • macOS (Homebrew): brew install --cask visual-studio-code
  • Linux (Snap): sudo snap install --classic code

    • Optional (macOS): enable the code shell command via Command Palette → “Shell Command: Install 'code' command in PATH”.

    Install from VSIX (GUI)

    Install LingTeX via CLI

  • Open Visual Studio Code.
  • Go to View → Extensions.
  • Click the ••• menu in the top-right of the Extensions panel.
  • Select “Install from VSIX…”.
  • Choose the downloaded LingTeX `.vsix` file.
  • When prompted, click “Install”. Reload the window if asked.
  • Missing dependencies: install “LaTeX Workshop” and “LaTeX Utilities”.
  • Biber missing: install biber (Step 1) for biblatex support.

    • If double-clicking the `.vsix` doesn’t open VS Code, use the steps above from inside VS Code.

    Install via CLI

    Verify your setup

    1. Open a LaTeX project in VS Code and run “LaTeX Workshop: Build with recipe → latexmk (xelatex)”.
    2. If your project uses biblatex, ensure the recipe runs biber (or run it manually between LaTeX passes).
    3. If you use glossaries (glossaries-extra), prefer \makenoidxglossaries (used by the sample template).

    Problems? Check the LaTeX Workshop “LOG & OUTPUT” pane for missing packages. Install any missing package using tlmgr (TeX Live/MacTeX) or your package manager.

    You can install the VSIX using the `code` command if it’s on your PATH.

    code --install-extension /path/to/LingTeX-x.y.z.vsix

    On macOS, you may need to enable the `code` shell command from Command Palette: “Shell Command: Install 'code' command in PATH”.

    Troubleshooting

    • VS Code says the extension is invalid: ensure the full `.vsix` downloaded and try again.
    • Missing dependencies: install “LaTeX Workshop” and “LaTeX Utilities” for the best experience.

    What’s included

    • TSV → Interlinear (gb4e) with spacing controls.
    • TSV ↔ LaTeX tabularx conversions; tabularx → Excel export.
